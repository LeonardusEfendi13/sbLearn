Basic - basic Springboot 1 (1/3)
Sebelum memulai sebuah aplikasi menggunakan framework Springboot, kita perlu menyiapkan beberapa dependency. Dependency merupakan sebuah modul eksternal yang memuat fungsi-fungsi tertentu untuk mendukung pengembangan aplikasi. Pada framework Springboot, dependency –dependency tersebut dikelola oleh build manager Maven atau Gradle. Pada direktori framework Springboot, seluruh dependency akan disimpan pada file pom.xml apabila menggunakan build manager Maven, dan disimpan pada file build.gradle apabila menggunakan build manager Gradle. Beberapa dependency yang umum digunakan diantaranya adalah

1. Dependency untuk membuat aplikasi web

2. Dependency untuk akses database menggunakan JPA (Java Persistence API)

3. Dependency untuk akses database MySQL

Basic - basic Springboot 1 (2/3)
Annotation pada Springboot merupakan sebuah tanda yang menggunakan “@” sebagai prefix yang dimana tanda ini mengharuskan sebuah komponen atau class pada springboot berperilaku sesuai dengan tandanya. Penggunaan annotation pada Springboot mempermudah pengembang untuk mengelola kode ataupun menghemat penulisan kode sudah terwakilkan oleh adanya annotation ini. Beberapa contoh penggunaan annotation pada springboot adalah sebagai berikut :

1. Annotation untuk menandakan class utama pada aplikasi.

2. Annotation untuk melakukan injeksi dependency secara otomatis sesuai dengan dependency pada class atau komponen yang diberikan annotation.

3. Annotation untuk memberikan handler (Post / Get) terhadap request HTTP yang masuk ke dalam endpoint.

Basic - basic Springboot 1 (3/3)
Untuk memulai sebuah aplikasi menggunakan framework Springboot, kita hanya perlu masuk ke website https://start.spring.io/ yang dimana halaman website tersebut akan terlihat seperti ini :


Pada bagian “Project”, kita perlu menentukan build manager apa yang akan digunakan untuk aplikasi kita, terdapat beberapa pilihan diantaranya “Gradle-Groovy”, “Gradle-Kotlin”, dan “Maven”.

Pada bagian “Language”, kita dapat menentukan bahasa pemrograman untuk aplikasi kita, baik Java, Kotlin, maupun Groovy”.

Pada bagian “Spring Boot”, kita akan menentukan versi framework Springboot yang akan digunakan sesuai keinginan kita.

Pada bagian “Project Metadata” akan mengisi detail dari setiap field seperti ingin membuat project dalam Java, dan diakhir akan menentukan versi Java sesuai keinginan.

Setelah semua terisi dan terpilih, langkah selanjutnya adalah menambah dependency yang diperlukan sesuai kebutuhan aplikasi.

Langkah terakhir adalah menekan tombol “Generate” dan framework Springboot akan terunduh dalam format Zip, kemudian kita hanya perlu mengekstrak file tersebut dan membukanya menggunakan IDE masing-masing.
